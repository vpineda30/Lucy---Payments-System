Sistema de Pagamentos com Typescript, Express e MySQL

Descrição do Projeto: O projeto tem como principal objetivo gerenciar e efetuar pagamentos fictícios ente usuários.

Arquitetura do Projeto: 

Esse projeto foi desenvolvido usando uma organização Monolitica Modular, uma arquitetura Limpa (Clean Architecture) e princípios fundamentais pra uma aplicação escalável, como SOLID. 
Os Módulos seguem um mesmo padrão, sendo uma pasta App (Use Cases, Entidades e gateways), Adapters (Controllers, Rotas e Middlewares) e Infra (Repositories e Providers), além de uma pasta compartilhada entre todos os módulos (Shared)
